7.和常规版本的差异
　　服务器专版的开发和以往的常规版本相比，有很多地方存在着差异，以下逐个说明。

1.func能调用的函数范围
　　大部分函数仍然生效，就象您在js中书写的那样可以正常被调用，但导致弹出对话框的函数将失效，同样，报表中导致弹出对话框的工具条功能号也将失效.
　　某些和界面交互有关的函数将无效，例如focusIE设置焦点、AddHot设置热键、打印预览等等;

2.界面的错误
　　界面中的错误将不会被弹出，包括报表在计算过程中的错误消息子窗口。但这些错误信息将会被捕捉并自动输出到临时目录下的"@ErrorLog.txt"文件中;

3.OnEvent事件
　　由于不采用页面容器，自然就没有事件了，OnReady、OnEvent都已经不存在;

4.reg.ini
　　reg.ini必须位于硕正服务器专版的DLL的同一个目录下才能生效，如果您开启了httplog，那么日志文件将位于临时目录下，文件名也变成了"@httplog.htm";

5.supcan.xml
　　supcan.xml仍然能生效，但不是从URL下载，如果您需要这个文件，就必须把它部署在硕正服务器专版的DLL的同一个目录下;

6.报表的异步计算
　　异步计算已无效，统一变成同步计算;

7.报表属性
　　报表属性中，“报表打开后就自动取数、填充”选项，在服务器专版中无效;

8.报表创建参数
　　部分创建参数已失效，比如自定义报表工具条，服务器专版已经没有工具条的概念;
